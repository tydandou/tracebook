# Automatic Creation Rules

For a newly discovered project, create only `index.md` and
`project-status.md`. Do not pre-create empty architecture, API, database,
business-rule, source-map, decision, log, or synthesis files.

Create category documents only when a verified conclusion needs them. Split a
long source map, business rules, API, or database document into module or
domain child documents after it exceeds 300 lines. Keep uncertainty marked as
`Pending` and never invent missing facts.
