# Source Attribution Rules

The knowledge base is not the primary source of truth. Prefer source code,
raw material, interface definitions, migrations, and test results. Add source
references for numbers, enums, API and database fields, business conditions,
core rules, architecture conclusions, and bug root causes.

Prefer `path/to/file.ext:L20-L38`. When no evidence can be verified, write
`Pending`. Do not promote an AI inference to a confirmed fact.
