# Synthesis Page Rules

Create a synthesis page only when a complex issue recurs, requires evidence
from multiple documents or modules, involves risk or state relationships, or a
user explicitly asks for a reusable conclusion.

Every synthesis page includes scope, current conclusion, key evidence and
sources, generation or update time, pending items, and invalidation
conditions. Do not make unsourced secondary summaries or generate one for all
material. Mark synthesis pages for review when their underlying facts change.

Project synthesis pages live under the resolver-provided project directory's `synthesis/` folder. Decide
whether cross-project synthesis belongs in `02-domain` or `03-patterns`.
